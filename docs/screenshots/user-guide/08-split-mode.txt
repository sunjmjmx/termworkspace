=== 场景：分屏模式（终端 + AI 同面板） ===
文件：08-split-mode.png（待补充真实截图）

触发方式：在当前 Tab 中点 🤖 按钮从终端切换到 AI 面板，或点 🖥️ 从 AI 切回终端

界面变化：
```
┌──────────────────────────────────────────────┐
│  ~/projects/termworkspace  Tab1 │ + │    🌙  │
├──────────┬───────────────────────────────────┤
│           │  ┌─── AI 对话面板 ───────────┐   │
│  src/     │  │                            │   │
│  docs/    │  │ 用户: 这个报错什么意思     │   │
│  tests/   │  │                            │   │
│           │  │ AI: Error: ENOENT 表示... │   │
│           │  │                            │   │
│           │  │ ┌──────────────────────┐   │   │
│           │  │ │ 输入消息...          │   │   │
│           │  │ └──────────────────────┘   │   │
│           │  └────────────────────────────┘   │
└──────────┴───────────────────────────────────┘
```

交互说明：
- 同一 Tab 内，终端和 AI 面板共享同一个位置
- 切换时终端进程不中断（后台运行）
- 适合：一边跑测试，一边问 AI 关于报错的问题
- 右上角按钮状态：终端模式显示 🤖，AI 模式显示 🖥️

代码路径：src/renderer/components/SplitPane.tsx

测试命令：启动 → 输入 npm test → 点 🤖 → 问 AI 关于测试输出 → 点 🖥️ 回到终端
测试结果：✓ 切换流畅，终端进程在 AI 对话期间保持运行
