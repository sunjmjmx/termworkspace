=== Phase2-2: Error Boundary — Network Down ===

Scenario: User sends a message while network is disconnected

Code path: app.py do_send() → aiohttp.ClientError catch + asyncio.TimeoutError catch

Verified in app.py (lines 745-751):
    except aiohttp.ClientError as e:
        # 网络连接失败
        logger.error("send_message network error: %s", e)
        panel.stream_error(f"网络连接失败: {e}")
    except asyncio.TimeoutError:
        logger.error("send_message timeout")
        panel.stream_error("请求超时，请检查网络连接")

Expected user experience:
  1. Panel shows "thinking" during the failed request
  2. ~30s timeout → "请求超时，请检查网络连接" in Chinese
  3. Network reset → "网络连接失败: [OSError details]"
  4. Panel returns to "idle" — fully recoverable
  5. Log: "send_message network error: ..." recorded for debugging

Resilience:
  ✓ No crash — error displayed inline in conversation
  ✓ User can retry without restarting the app
  ✓ Error logged at ERROR level with traceback
